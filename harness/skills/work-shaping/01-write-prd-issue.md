# Write PRD Issue

## When to use

Use this after the request is clear enough to become an executable work item.

## Agent prompt

Create or update a GitHub Issue that works like a small PRD. The issue is the active work object; product wiki truth changes only when cadence applies the accepted issue.

## Steps

1. State the goal, user value, and non-goals.
2. Name target wiki node ids and any new wiki nodes that must be created from templates.
3. Capture proposed wiki wording, behavior, or section replacements.
4. Add acceptance criteria in product language.
5. Add QA expectations, screenshot or flow-tree evidence needs, and edge cases.
6. Name likely code targets only as a first guess.
7. Add labels and dependencies so the issue can be picked up by one agent or split later.

## Done

- A GitHub Issue names target and new wiki nodes.
- The issue has PRD-like acceptance criteria.
- The issue is ready to be accepted, split, or picked up by cadence.
