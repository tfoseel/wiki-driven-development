# Grill Request

## When to use

Use this when a user request is still vague, broad, or missing product intent.

## Agent prompt

Turn a loose request into crisp product intent before creating a GitHub Issue. Ask only for information that changes scope, acceptance criteria, or target wiki nodes.

## Steps

1. Restate the requested outcome in product language.
2. Identify the affected user, workflow, and business rule.
3. Ask the smallest useful clarification if target wiki nodes or acceptance criteria are unclear.
4. Separate goals, non-goals, constraints, and open questions.
5. Decide whether the request is ready for a PRD-shaped issue.

## Done

- The request has a clear product outcome.
- Unknowns are explicit.
- The next step can write or update a PRD-shaped GitHub Issue.
