# QA Nodes

## When to use

Use this for executable coverage, edge cases, and verification scope.

## Authoring contract

- Do not repeat the full flow narrative.
- Define coverage model, seeds, happy paths, edge cases, and failure states.
- Reference test files and commands in metadata.
- Include scenarios in given/when/then form.
- Link screens, flows, actions, policies, and models under test.

## Evidence

QA nodes are verified by test output and refreshed visual evidence on affected screen and flow nodes.
