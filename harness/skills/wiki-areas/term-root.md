# Term And Root Nodes

## When to use

Use this for the product overview, glossary, index pages, and non-generated knowledge.

## Authoring contract

- Keep root pages as navigation and product framing, not task logs.
- Define terms once and link nodes that use them.
- Avoid implementation churn unless it changes product understanding.
- Keep stable machine references in hidden WDD metadata.
- Link core entities, actions, screens, flows, policies, design, and QA.

## Evidence

Root and term nodes usually use `code: not_required`; consistency is verified by links, metadata, and ready checks.
