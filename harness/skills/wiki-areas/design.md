# Design Nodes

## When to use

Use this for product UI tokens, component behavior, visual states, and accessibility rules.

## Authoring contract

- Describe product UI rules, not the wiki reader or harness UI.
- Define tokens, density, layout rules, control states, and accessibility requirements.
- Link screens that consume the design contract.
- Keep generated or reference images as artifacts when useful.
- Update QA when visual state is part of product behavior.

## Evidence

Design changes are verified by screen screenshots, visual review, or automated visual checks when available.
